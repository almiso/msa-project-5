# Задание 3. Реализация Distributed Scheduling с k8s CronJob

В данном задании реализована выгрузка аналитических данных об отгрузках (таблица `shipments`) из локальной логистической БД с использованием **Kubernetes CronJob**. Расписание (CRON) настроено на запуск ежедневно в 20:00.

## Описание технического решения 

Подготовлен скрипт для простого экспорта данных из целевой таблицы БД, а также манифесты для тестирования и деплоя:

1.  **Скрипт выгрузки (`exporter/export.sh`)**
    Легковесный bash-клиент, извлекающий доступы к БД из переменных окружения (`envVar`). Скрипт использует потоковую инструкцию БД `psql \copy` для высокоскоростного (streaming) экспорта выборки в CSV-формат. Утилита выводит статистику по байтам и отрывок файла в `stdout` для сбора лог-агентами (Fluentd/Promtail) кластера Kubernetes.
2.  **Контейнеризация (`exporter/Dockerfile`)**
    Образ базируется на минималистичном `alpine:3.19`. За счет отказа от тяжелых библиотек языков программирования и установки лишь `postgresql-client` — время холодного старта образа занимает доли секунды.
3.  **Оркестрация (`k8s/cronjob.yaml`)**
    Основной манифест пакетного планировщика. Определяет расписание (CRON: `0 20 * * *`), жесткие лимиты выделенной памяти (`128Mi`) и процессорного времени (`200m`), а также политику устойчивости к сбоям (`restartPolicy: OnFailure` и `concurrencyPolicy: Forbid`).
4.  **Мок инфраструктуры (`k8s/postgres.yaml`)**
    Инфраструктурный Deployment с тестовой СУБД для демонстрации. С помощью ConfigMap инициализируются DDL-миграции таблицы `shipments` и автоматический сид случайных фейковых данных (Трек-коды, пункты отправки и статусы логистики).

## Демонстрация результата

Результаты успешного запуска CronJob (в кластере `minikube`) наглядно зафиксированы на приложенном скриншоте. Консольный вывод подтверждает безопасное подключение пода к БД, создание валидного файла структуры CSV и имитацию отправки массива аналитикам. Контейнер успешно выполняет задачу и корректно отключается со статусом пода `Completed`.
